# Hodo — project briefing

Hodo is a portfolio-quality AI travel planner (Next.js 16 App Router, React 19, TypeScript, Tailwind v4, Zustand). An 8-step wizard (or plain chat) collects the trip; Claude builds a day-by-day itinerary that **constructs itself live on screen**, then you drag/drop, edit inline, lock cards, and price it. Also: multi-currency live budget, Leaflet map, Open-Meteo weather, live hotel/flight prices, undo/redo, day quick actions (⋯), read-only share links, export (PDF/Markdown/copy), Checklist + Reservations tabs, a "Book this trip" checklist stage, a mobile-first Day-of travel mode ("Today" tab), one-time post-build coach marks. The user journey: **plan → refine → book → travel**. The owner is non-technical — write the code, run the commands, explain in plain English, and ask before anything irreversible or money-costing.

## Architecture

- **Data model** (`src/lib/types.ts`): `TripPlan { days[], budget, preferences, assumptions?, liveData?, checklist?, reservations? }`; `Activity { cost, locked, … }`. One Zustand store (`src/lib/store.ts`, `persist` + `skipHydration`, key `wandr-v1`); ALL trip mutations go through store actions — agent and UI share the same paths. Cross-tree events use a `wandr:*` DOM bus owned by the always-mounted `ChatBridge` in AppShell (listeners in closable panels silently no-op — learned the hard way).
- **Entry + generation**: the wizard (`src/components/wizard/*`, `src/lib/wizard.ts`, static datasets in `src/lib/data/*` — zero AI) maps answers to `draftPreferences` + ONE `composeWizardMessage` send; answered fields become `message`-sourced assumption chips, skipped ones `inferred` (dotted). The agent returns **structured JSON patches** after a `---WANDR-JSON---` marker; **patches MERGE by day id — never replace the `days` array**. New trips: **chunked skeleton-first** (empty-days skeleton, then 3-day fill batches), streamed with 10s heartbeats + 30s client watchdog, `maxDuration = 300` (needs Fluid Compute). **Model tiering** by `intent`: Sonnet 4.6 (`full`) / Haiku 4.5 (`quick`); thinking disabled. Fill batches send already-built days as one-line summaries (token diet); user free text in the system prompt is `<user_text>`-delimited (injection guardrail).
- **Live reveal** (`src/hooks/useRevealSequencer.ts`, timing tokens in `src/lib/revealTiming.ts`): a presentation queue decouples batch arrival from display — days reveal strictly in order with animated entrances, driven ONLY by real pipeline events (never simulated progress). The sequencer owns `endBuild()`; chat stays closed during/after a build (unread dot on the Chat button holds the agent summary).
- **Provider wrapper** (`/api/live-prices`, `src/lib/providers/*`): liteAPI (hotels) + Travelpayouts/Aviasales (flights, "indicative"). **Graceful degradation**: missing key/failure → AI estimates, app never breaks. Cached on `trip.liveData`, refetched only when destination/dates/origin/accommodation change.
- **Currency** (`src/lib/currency.ts`): convert **at the aggregation boundary** via live ECB rates (Frankfurter, 24h cache, hardcoded fallback); `budget.currency` is the single source of truth, local price shown muted.
- **Locked cards**: system-prompt rule + client backstop (`preserveLockedActivities`); manual edits auto-lock. Locked = protected from AI only.
- **Undo/redo** (store history slice): state-level — each qualifying mutation pushes the pre-mutation trip **object reference** (immutable updates → structural sharing; cap 50, per trip). Agent patches capture one entry per applied response ("AI updated Day 4"). Surfaces: Ctrl/Cmd+Z / Shift+Z (inert while typing), header buttons w/ label tooltips, Undo toasts on deletes/patches. Disabled during builds; stack resets at settle. **Limitation: in-memory only — refresh clears it** (deliberate).
- **Day quick actions** (`DayQuickMenu.tsx`): ⋯ menu dispatches canned instructions through the existing quick-tier chat pipeline (real user messages) — locks/undo/toasts/retry apply automatically; `store.quickActionDayId` drives the per-day busy state.
- **Share links** (`src/lib/shareStore.ts`, `/api/share`, `/t/[id]`): immutable snapshot under an unguessable 12-char id in **Vercel Blob** (in-memory fallback locally; on Vercel without Blob it returns an honest 503 — a fallback "success" would be a dead link). `/t/{id}` is public (proxy-exempt), server-rendered, strictly read-only, deep links verbatim (affiliate markers intact), OG tags, graceful 404. Re-sharing mints a new id.
- **Book this trip** (`src/lib/booking.ts` pure lib, `BookingPanel.tsx`): a checklist **derived** from trip state — flight legs (origin↔cities + inter-city), one hotel stay per consecutive-nights city cluster (cities inferred by **geographic day-centroid clustering**, 60km threshold — there is no explicit per-day city in the data model), and only reservation-worthy activities (`bookingUrl` or reservation-ish language). Affiliate deep links carry markers as fetched; prices labelled "indicative"; disclosure always visible; nothing gated behind booking. "Mark as booked" creates a linked entry in the EXISTING Reservations machinery (budget actuals + itinerary badges follow free); booked/skipped live in `trip.bookingStatus` so the normal undo snapshots cover them.
- **Day-of mode** (`src/lib/dayOf.ts` pure lib, `TodayPanel.tsx`): when today falls inside the trip dates, an on-trip banner appears and the "Today" tab leads the row (otherwise it sits last as a preview). Mobile-first, offline-graceful (persisted local state, no AI/network), local currency LARGE with budget conversion small, Google Maps geo links from stored lat/lng, mark-as-done is purely visual (`trip.dayOfDone`, undoable) and recalculates remaining-today. The clock has a test seam: `globalThis.__HODO_NOW__`. **No notifications, GPS, or background behaviour — deliberate; future options.**
- **Overrides**: `PLANNER_MODEL` / `QUICK_MODEL` env vars re-tier the models.

## Standing rules

1. **Definition of done** — every feature/fix ends with unit tests (`npm run test:unit`, vitest over the pure libs) AND the full Playwright suite passing: (a) on a **local production build** (`npm run build` + `next start`) before "done"; (b) on the **live Vercel deployment** (`VERCEL_BYPASS=<token> BASE_URL=… npx playwright test`) before "shipped". Extend the suite for new behaviour. CI (GitHub Actions) runs build + units + the mocked subset on every push. (Real long-trip test: `RUN_LONGTRIP=1`.)
2. **Evidence-first debugging** — reproduce with a test or probe and watch it fail *before* fixing; instrument rather than theorize. Never guess-and-patch.
3. **Completion contract** — not done until actually complete (days-filled == days-promised). Partial/failed states must be **visibly partial** with a retry path, never a silent success.

## Environment / providers

Server-side only, in `.env.local` and Vercel env vars for **Production + Preview + Development** (recurring gotcha: Preview unticked → key missing):
`ANTHROPIC_API_KEY` (required — API credits, NOT the Claude subscription) · `DEMO_PASSWORD` (optional gate — pages AND `/api/*`, 401 JSON without cookie) · `LITEAPI_API_KEY` (hotels) · `TRAVELPAYOUTS_TOKEN`+`_MARKER` (flights — not yet provisioned) · `BLOB_READ_WRITE_TOKEN` (share links; auto-added when Blob storage is enabled, access **public**) · optional `PLANNER_MODEL`/`QUICK_MODEL`. All AI/provider/share routes are per-IP rate-limited (`src/lib/rateLimit.ts`). Vercel: Deployment Protection on (tests plant a bypass cookie); Fluid Compute required for 300s. Keyless: Open-Meteo, OpenStreetMap, Frankfurter.

## Known patterns

- **Silent failures are the recurring bug species** — "Enter reloads chat" (silent stream gap), "10 days got 3" (patch replaced instead of merged), share links that "worked" but 404'd for recipients (per-instance memory on serverless), chat-event listeners unmounting with the chat panel. Always surface failure/partial state explicitly and make it recoverable.
- **Agent output is validated with one retry** — tolerant JSON extraction (balanced-object), then one self-correcting retry with the error fed back before surfacing an incomplete state.
- **Two environmental traps**: custom CSS classes in `globals.css` out-cascade Tailwind utilities of equal specificity (position/display conflicts render elements zero-size); Next bundles API routes and pages separately, so module-scope singletons must live on `globalThis` to be shared within a process.
